# Промпт для улучшения эстетики фронтенда

## Контекст проекта

Это SaaS продукт для виртуального стайлинга интерьеров с помощью AI. Лендинг должен выглядеть профессионально, современно и вызывать доверие у целевой аудитории (риелторы, агенты по недвижимости).

## Текущее состояние

**Что уже есть и работает хорошо:**
- Четкая структура и информационная архитектура
- Хорошая читаемость и UX
- Функциональные интерактивные элементы (before/after slider)
- Адаптивный дизайн

**Что можно улучшить (без кардинальных изменений):**
- Типографика: используется Inter (слишком распространенный шрифт)
- Цветовая палитра: фиолетовый градиент (классический "AI slop")
- Анимации: минимальные, только hover эффекты
- Фоны: простые solid цвета и базовые градиенты

## Промпт для улучшения

### Общий подход

При улучшении существующего дизайна лендинга VirtualStagingAI:

**Принцип:** Улучшать, а не переделывать. Сохранять структуру, логику и общую концепцию, но повышать визуальное качество через точечные улучшения.

### 1. Типографика (улучшение без изменения структуры)

**Текущее:** Inter (слишком распространенный, generic)

**Улучшение:**
- Заменить Inter на более выразительный, но читаемый шрифт
- **Рекомендации для SaaS/недвижимости:**
  - **Satoshi** или **Cabinet Grotesk** - современные, профессиональные, но не перегруженные
  - **IBM Plex Sans** - техничный, но дружелюбный
  - **Source Sans 3** - чистый, универсальный
- Сохранить текущие размеры и веса, но использовать более контрастные комбинации (например, 300/400 для body, 700/800 для заголовков вместо 400/600)
- Добавить больше воздушности между строками (line-height) для улучшения читаемости

**Не делать:**
- Не менять размеры заголовков кардинально
- Не использовать декоративные шрифты, которые ухудшат читаемость
- Не нарушать текущую иерархию текста

### 2. Цветовая палитра (эволюция, а не революция)

**Текущее:** Фиолетовый градиент (#667eea → #764ba2)

**Улучшение:**
- Сохранить фиолетовую основу, но сделать более уникальной
- **Варианты улучшения:**
  - Более глубокий, насыщенный фиолетовый с акцентами (например, добавить темно-синий или индиго)
  - Или перейти к более профессиональной палитре: глубокий синий + золотой акцент (символизирует премиум качество)
  - Или использовать более теплую палитру: терракотовый + кремовый (ассоциируется с домом, уютом)
- Использовать CSS переменные для консистентности
- Добавить более выразительные акцентные цвета для CTA кнопок

**Не делать:**
- Не менять полностью цветовую схему (это сломает брендинг)
- Не использовать слишком яркие, кричащие цвета
- Не терять контрастность для accessibility

### 3. Анимации и микро-взаимодействия

**Текущее:** Только hover-scale эффекты

**Улучшение:**
- Добавить плавные появления элементов при скролле (fade-in, slide-up)
- Улучшить переходы между состояниями (более плавные, с easing функциями)
- Добавить subtle анимации для иконок и карточек
- Использовать CSS-only решения (не добавлять тяжелые библиотеки)
- Сфокусироваться на высоко-влиятельных моментах:
  - Появление hero секции при загрузке
  - Плавное появление карточек в галерее
  - Анимация before/after slider (уже есть, можно улучшить)

**Не делать:**
- Не добавлять слишком много анимаций (это отвлечет)
- Не использовать тяжелые JavaScript библиотеки
- Не делать анимации слишком быстрыми или медленными

### 4. Фоны и атмосфера

**Текущее:** Простые solid цвета (bg-gray-50, bg-white) и базовый градиент

**Улучшение:**
- Добавить глубину через subtle градиенты или текстуры
- Использовать более сложные градиенты для hero секции (не просто линейный)
- Добавить subtle паттерны или геометрические элементы для фона
- Использовать backdrop-filter более эффективно (уже используется в nav)
- Добавить легкие тени и depth для карточек

**Не делать:**
- Не делать фоны слишком сложными (это отвлечет от контента)
- Не использовать слишком контрастные паттерны
- Не терять читаемость текста на фоне

### 5. Компоненты и детали

**Улучшения без изменения структуры:**
- Улучшить тени карточек (более реалистичные, многослойные)
- Добавить более выразительные border-radius для некоторых элементов
- Улучшить spacing между элементами (больше воздуха)
- Добавить subtle градиенты на кнопках вместо solid цветов
- Улучшить иконки (возможно, использовать более выразительные варианты)

## Изолированные промпты для точечных улучшений

### Только типографика
```
Заменить Inter на более выразительный шрифт для SaaS продукта в сфере недвижимости.
Использовать Satoshi или Cabinet Grotesk. Сохранить текущие размеры, но использовать
более контрастные веса (300/400 для body, 700/800 для заголовков). Улучшить line-height
для лучшей читаемости. Не менять структуру и размеры заголовков.
```

### Только цветовая палитра
```
Эволюционировать фиолетовый градиент в более уникальную и профессиональную палитру.
Сохранить фиолетовую основу, но сделать более глубокой и насыщенной. Добавить акцентные
цвета (золотой или терракотовый) для CTA элементов. Использовать CSS переменные для
консистентности. Не терять контрастность и accessibility.
```

### Только анимации
```
Добавить плавные CSS-only анимации появления элементов при скролле. Улучшить переходы
между состояниями с правильными easing функциями. Сфокусироваться на hero секции и
карточках галереи. Не добавлять слишком много анимаций, сохранить производительность.
```

### Только фоны
```
Добавить глубину и атмосферу через subtle градиенты и текстуры. Улучшить hero секцию
более сложным градиентом. Добавить легкие геометрические элементы для фона. Не терять
читаемость и не отвлекать от контента.
```

## Что НЕ делать

- ❌ Не менять структуру HTML и layout
- ❌ Не переделывать компоненты полностью
- ❌ Не использовать слишком экспериментальные решения
- ❌ Не терять производительность
- ❌ Не нарушать accessibility
- ❌ Не использовать слишком много новых зависимостей
- ❌ Не менять функциональность интерактивных элементов

## Приоритет улучшений

1. **Высокий приоритет:**
   - Типографика (быстрое улучшение восприятия качества)
   - Цветовая палитра (уникальность бренда)

2. **Средний приоритет:**
   - Анимации (улучшение UX)
   - Фоны (атмосфера)

3. **Низкий приоритет:**
   - Детали компонентов (полировка)

## Использование

Этот промпт можно использовать:
- В системном промпте Claude для консистентных улучшений
- В файле CLAUDE.md в корне проекта (автоматически читается Claude Code)
- Для изолированных улучшений конкретных аспектов дизайна
